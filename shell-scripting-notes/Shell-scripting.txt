#)Special variables : 
---------------------

*)Special variables are predefined variables that give information about the script, arguments, and system.


All arguments(variables) of the script are the values you pass after the script name when running it.

| Variable | Purpose                                   |
| -------- | ----------------------------------------- |
| `$0`     | Name of the script                        |
| `$1`     | Displays First argument                   |
| `$2`     | Diaplays Second argument                  |
| `$#`     | Displays count of Total  arguments        |
| `$@`     | All arguments (separately)                |
| `$*`     | All arguments (as one string)             |
| `$?`     | Exit status of previous command (0 = success) |
| `$$`     | Process ID of the script or command       |
| `$!`     | Process ID of last background command     |


****)Linux exit status range: 0–255

1)0 = success

2)Non-zero = fail

1–255(non-zero) = different types of errors

#)REDIRECTIONS & LOGS:
----------------------

*)Redirections are used to send command output(either SUCCESS or FAILED i.e errors) to a log-file instead of displaying on the screen.
 
*) 1 --> SUCCESS
*) 2 --> FAILED
*) & --> SUCCESS/FAILED
*) > --> overwrite the data
*) >> --> append the data

LOGS: Logs are records of what happens during the execution of an action and used for monitoring and troubleshooting.
-----


Examples:
----------


*) ls -l 1> output.log --> it stores the command result in "output.log" only when it is success

*) lsdfgg -l 2> output.log --> it stores the command result in "output.log" only when it is failure

*) ls -l &>> output.log --> it stores the command result in "output.log" irrespective of the failure or success.


Example-1: Assume your script file name is "Task.sh"
----------

First you need to create a log directory inside "/var/log" which is inside root user to store the output of a script or command,

Assume you want create a "shell-script-logs" folder, as you are working as non-root user so first create the folder with sudo permissions

sudo /var/log/shell-script-logs 


*)| tee -a $LOG_FILE --> appends the data to the log file and also display the messages on the screen 

*)example: echo " nginx is already exists...SKIPPING " | tee -a $LOG_FILE



#!/bin/bash 

USER_ID=$(id -u)

LOG_FOLDER="/var/log/shell-script-logs"

SCRIPT_NAME=$(echo $0 | cut -d "." -f1)  # it will remove the .sh so the script name will be "Task"

mkdir -p $LOG_FOLDER

echo " Script started and executed on : $(date) "

if [ $USER_ID -ne 0 ]; then
	echo " ERROR:: please run the script with user privileges "

	echo " $2 installation is.. SUCCESS "
fi

Cron Job Format:
----------------

This Cron Job format has 5 fields which are stars to define the schedule 

* * * * * command/script 
- - - - -
| | | | |
| | | | +---- Day of Week (0-7) or (SUN-SAT)
| | | +------ Month (1-12)
| | +-------- Date of Month (1-31)
| +---------- Hour (0-23)
+------------ Minute (0-59)

# TIPS:
-------

*          → every value
,(comma)   → multiple specific values
-(hyphen)  → range / between values
*/         → repeat at intervals



*) * * * * * /home/ec2-user/test.sh --> script runs every minute 

1) */10 * * * *
   → every 10 mins, everyday

2) 0 */2 * * *
   → every 2 hours at 0th minute everyday

3) 0 9-21 * * *
   → runs at 0th minute from 9 AM to 9 PM everyday

   Meaning:
   0    → 0th minute
   9-21 → 9 AM to 9 PM

   Runs at:
   9:00
   10:00
   11:00
   ...
   21:00

4) * 6,17 * * *
   → runs every minute during 6 AM and 5 PM hours

   Runs at:

   6:00 AM
   6:01 AM
   ...
   6:59 AM

   5:00 PM
   5:01 PM
   ...
   5:59 PM

5) 0 2 * * MON-FRI
   → runs Monday to Friday at 2 AM only

6) 0 4 * * MON,THU
   → runs on Monday and Thursday at 4 AM only

7) * * * * *
   → runs every minute

8) * 9-18 * * *
   → runs every minute from 9 AM to 6 PM

9) * * * * MON
   → runs every minute on Monday



*   → Every value / Any value

      Example:
      * in hour field = every hour

,   → Multiple specific values

      Example:
      MON,WED,FRI
      = Monday, Wednesday, Friday

-   → Range of values (from - to)

      Example:
      9-18
      = from 9 AM to 6 PM

*/  → Repeating interval

      Example:
      */10
      = every 10 units
      (every 10 mins if used in minute field)

##)Running Shell Script Like a Command:
---------------------------------------

*)Any scripts that are moved to  PATH/bin folders considered as a command. you can run the script names like ls, pwd, ls -l commands

*)Normally shell scripts are created like:

backup.sh
cleanup.sh
test.sh


Step-1:
--------

Give execute permission to the script:

chmod +x backup.sh


Step-2:
--------

Move the script to a PATH location:

sudo mv backup.sh /usr/bin/


Step-3:
--------

Now the script can be executed as a normal Linux command like:

backup.sh

(OR)

In real-time, many users remove the ".sh" extension for better and cleaner command naming.

Example:
---------

Rename the script:

mv backup.sh backup


Then move it to PATH location:

sudo mv backup /usr/local/bin/


Now directly run like a Linux command:

backup


Real-Time Use:
---------------

Used for:
- automation scripts
- monitoring scripts
- backup utilities
- internal DevOps tools



Why Scripts Are Moved to /usr/local/bin/ :
-------------------------------------------

*)/usr/local/bin/ is a PATH directory in Linux.

PATH means:
------------
List of directories where Linux searches for commands.


Example PATH directories:
--------------------------

/bin
/usr/bin
/usr/local/bin
/sbin
/usr/sbin


When a script is moved to /usr/local/bin/ and given execute permission:

chmod +x backup


It can be executed as a normal Linux commands like ls, pwd, ls -l:

backup


Example:
---------

which ls

Output:
--------

/usr/bin/ls

Meaning:
---------

*)It tells in which directory the "ls" command is present.
